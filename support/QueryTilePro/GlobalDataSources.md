# Introduction 
If you are replicating or copying dashboards across team projects, copying and managing the underlying work item queries can become a short-term pain and a long-term administrative nightmare. Global Data Sources solve that problem by handling your query definitions in one central place.  
Global Data Sources can help you:
+ Minimize the effort of setting up, copying, and maintaining dashboards
+ Minimize the long term administrative cost of handling multiple queries across multiple team projects

# An example 
Let's say you want to copy a dashboard with 30 different widgets to 10 team projects. This can easily result in 10-30 queries copied to different projects, resulting in 100-300 queries.

To build the same dashboard using Global Data Sources you only define each source query once in a central place, and you can then copy widgets across boards or team projects. 

# Creating Global Data Sources  
The easiest way to create a global data source is by importing an existing query. To create a new Global Data Source, just go to the Query Tile PRO settings and click on the Data Source tab. 
From here you can create a new empty query or import an existing work item query as a global data source.
